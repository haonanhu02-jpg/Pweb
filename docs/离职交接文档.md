## 1. 项目概况

本项目是一个国央企招标公告采集与弹簧采购需求识别 POC，项目目录为 `D:\Pweb\mix采集poc`。系统目标是从公开招标/采购平台采集公告，标准化入库，下载并解析附件，识别是否存在可销售的弹簧、弹簧组件、弹性件或相关备件采购需求，并通过本地 Web 后台进行人工复核。

当前范围只覆盖公开页面采集，不做自动投标、自动报价、供应商账号登录、验证码处理、CRM 销售跟进或外网生产化部署。

## 3. 系统架构

系统分为五层：

1. 数据来源层：公开招标公告平台和央企自有采购公开栏目。
2. 采集层：Python CLI + `BaseSpider` 模板，各平台 Spider 只处理列表页、详情页和字段差异。
3. 数据处理层：标准公告模型 `TenderNoticeV1`、SQLite 入库去重、原始 HTML 留存、附件下载与解析。
4. 业务筛选层：规则引擎识别弹簧采购需求，输出 `high`、`review`、`low`、`excluded`。
5. 业务系统层：FastAPI 后端 + React 前端，支持看板、列表、详情证据和人工复核。

## 6. 本地启动和验证

### Docker 方式

先启动 Docker Desktop，然后执行：

```powershell

```

访问地址：

- 前端后台：`http://localhost:5173`
- 后端健康检查：`http://localhost:8000/api/health`
- Crawlab：`http://localhost:8080`

```powershell
docker compose -f docker-compose.crawlab.yml ps
docker compose -f docker-compose.crawlab.yml logs -f crawlab
```

如果 Docker API 报 `dockerDesktopLinuxEngine` 不存在，通常是 Docker Desktop 未启动。

### Docker 容器说明

Compose 文件：`docker-compose.crawlab.yml`  
Compose 项目名：`tender-poc-crawlab`

| 服务 | 镜像/构建 | 端口 | 主要用途 |
| --- | --- | --- | --- |
| `mongo` | `mongo:4.2` | 不对外暴露 | Crawlab 元数据数据库。 |
| `crawlab` | `docker/crawlab/Dockerfile` | `8080:8080` | Crawlab 调度平台，用来运行现有 Python CLI 爬虫命令。 |
| `api` | `docker/api/Dockerfile` | `8000:8000` | FastAPI 后端，读取 SQLite 并提供看板、列表、详情、复核接口。 |
| `web` | `docker/web/Dockerfile` | `5173:5173` | React 前端工作台。 |

关键挂载：

| 宿主机路径 | 容器路径 | 说明 |
| --- | --- | --- |
| `./:/opt/tender-poc:ro` | `/opt/tender-poc` | Crawlab 容器只读挂载项目源码，避免容器改源码。 |
| `./data:/data/tender-poc` | `/data/tender-poc` | 采集数据共享目录，SQLite、HTML、附件、导出都在这里。 |
| `./docker-data/mongo:/data/db` | `/data/db` | MongoDB 持久化数据。 |
| `./docker-data/crawlab:/root/.crawlab` | `/root/.crawlab` | Crawlab 本地配置和运行数据。 |

关键环境变量：

| 变量 | 值 | 说明 |
| --- | --- | --- |
| `TENDER_POC_DATA_DIR` | `/data/tender-poc` | 容器内统一数据目录。 |
| `PYTHONPATH` | `/opt/tender-poc/src` | 让容器内可直接运行 `python -m tender_poc ...`。 |
| `CRAWLAB_NODE_MASTER` | `Y` | 当前是单节点 Crawlab Master。 |
| `CRAWLAB_MONGO_HOST` | `mongo` | Crawlab 连接 Compose 内的 Mongo 服务。 |
| `VITE_API_BASE_URL` | `http://localhost:8000` | 前端访问后端 API 的地址。 |

常用容器命令：

```powershell
cd D:\Pweb\mix采集poc

# 启动或重建
docker compose -f docker-compose.crawlab.yml up -d --build

# 查看容器状态
docker compose -f docker-compose.crawlab.yml ps

# 查看日志
docker compose -f docker-compose.crawlab.yml logs -f crawlab
docker compose -f docker-compose.crawlab.yml logs -f api
docker compose -f docker-compose.crawlab.yml logs -f web

# 容器内验证 Python CLI
docker compose -f docker-compose.crawlab.yml exec crawlab sh -lc "cd /opt/tender-poc && python -m tender_poc stats"

# 停止服务但保留数据
docker compose -f docker-compose.crawlab.yml down
```

容器运行注意事项：

- Docker Desktop 必须先启动。
- 当前 SQLite 仍是 POC 存储，Crawlab 多个 Spider 定时任务需要错峰运行，避免并发写库。
- `data/` 和 `docker-data/` 是本机运行数据，不在 GitHub 仓库中；换机器部署时需要单独迁移。
- 如果端口冲突，优先修改 `docker-compose.crawlab.yml` 的宿主机端口，例如把 `5173:5173` 改成 `15173:5173`。

## 8. Crawlab 交接说明

Crawlab 只负责调度现有 CLI 命令，不包含业务逻辑。已有平台命令都可以直接作为 Spider 命令使用：

```bash
cd /opt/tender-poc && python -m tender_poc run ggzy --limit 20
cd /opt/tender-poc && python -m tender_poc run sdic --limit 20
cd /opt/tender-poc && python -m tender_poc run cec --limit 20
cd /opt/tender-poc && python -m tender_poc run crrc --limit 20
cd /opt/tender-poc && python -m tender_poc run chnenergy --limit 20
```

建议错峰调度，避免 SQLite 并发写入：

| Spider | 建议 Cron |
| --- | --- |
| tender-ggzy | 0 */2 * * * |
| tender-crrc | 10 */2 * * * |
| tender-sdic | 20 */2 * * * |
| tender-chnenergy | 30 */2 * * * |
| tender-cec | 40 */2 * * * |

注意：`chnenergy` 命令已经在容器中验证可以运行，但 Crawlab UI 中是否已经创建对应 Spider 和定时任务，需要交接人登录 UI 复核。

## 9. 数据模型和去重规则

核心表：

| 表 | 说明 |
| --- | --- |
| tenders | 标准公告主表，保存标题、平台、采购人、发布时间、正文、附件列表、raw_fields、content_hash。 |
| attachment_documents | 附件下载和解析结果，保存附件 URL、文件路径、解析状态、正文和错误信息。 |
| spring_demand_assessments | 弹簧需求识别结果，保存阶段、需求类型、等级、分数、命中词、证据和原因。 |
| spring_demand_reviews | 人工复核标记，保存 pending_review、relevant、invalid、viewed 和备注。 |

最小去重逻辑：

- `source_url/platform_url` 唯一。
- `content_hash` 唯一。
- 重复采集会计入 `duplicate_count`，不重复写库。

## 10. 弹簧需求识别逻辑

规则目标不是简单匹配“弹簧”两个字，而是判断公告是否存在可销售的弹簧、弹簧组件、弹性件或相关备件采购需求。

等级含义：

| 等级 | 含义 |
| --- | --- |
| high | 开放中的明确弹簧/弹簧组件/备件采购机会，优先跟进。 |
| review | 疑似相关，需要人工看原文或附件。 |
| low | 识别到需求线索，但可能是结果公告、开放性不足或价值较低。 |
| excluded | 未识别到可销售弹簧采购需求，或命中负向规则。 |

当前仅识别到 1 条 `low`，没有 `high/review`。这不是系统错误，主要原因是当前公开样本中缺少开放中的明确弹簧采购公告。

## 11. 已知问题和风险

1. 真实高价值订单通常在央企自有平台深层或登录区，公开官网栏目可能只有结果公告或少量公开信息。
2. 当前没有处理登录、验证码、CA、短信、供应商账号权限。
3. SQLite 适合 POC，多个 Crawlab 任务需要错峰，正式生产建议迁移 PostgreSQL/MySQL。
4. PowerShell 输出中文偶尔会乱码，通常是终端编码问题，数据库和前端 UTF-8 数据本身正常。
5. 附件解析不做 OCR，扫描版 PDF 会解析为空。
6. 当前 Web 未做登录鉴权，只适合本机或内网验证。
7. Docker Desktop 未启动时，`docker compose ps` 会报 Docker API 连接失败。
